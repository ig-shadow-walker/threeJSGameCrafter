# Concept Definition Of Done

A concept is ready to hand to `threejs-game-director` only when all of these are true:

- High concept stated in one sentence (genre, verb, objective, pressure, world).
- Player fantasy and a unique hook are named (not a bare clone).
- Three pillars are defined and are actually used as tie-breakers.
- Core loop stated in ONE sentence with all six parts: verb, objective, pressure, reward, fail state, restart.
- Every mechanic maps to a concrete input on BOTH desktop and mobile/touch.
- Win/end condition and difficulty ramp are defined.
- Required UI/HUD states are listed.
- Art direction and audio direction are described and are achievable/consistent.
- Asset map lists high-value surfaces, each tagged procedural / image-generator / 3D-generator / audio-generator / hybrid.
- Scope tiers defined: a vertical slice (build first), a target build, stretch goals, and explicit out-of-scope.
- Feasibility checked: nothing in the target build obviously breaks a browser real-time budget without a stated cheaper path.
- Top 3 risks listed, each de-risked by the vertical slice.
- GDD written to `docs/game-design.md` using the template.

If any item is missing, the concept is not ready — resolve it or mark it explicitly deferred before handoff.
